# Kisaan Roadmap

## Phase 1: Validate the Workflow

Goal: confirm the exact service and payment behavior of real providers.

- interview 3 to 5 machinery service providers
- map common service types
- understand how rates are set
- understand how partial payments happen
- note where disputes usually happen

## Phase 2: Manual Ledger Pilot

Goal: validate the workflow without expensive engineering.

- use sheets or a simple prototype
- track one provider and a few farmers
- test whether digital records reduce confusion
- identify which screens are used daily

## Phase 3: MVP Product Build

Goal: build a focused product that providers can use every day.

- provider login
- farmer creation
- service entry form
- ledger dashboard
- payment recording
- statement view

## Phase 4: Field Testing

Goal: learn from real-world use, not assumptions.

- onboard one provider fully
- observe daily usage for 2 to 3 weeks
- reduce friction in entry flow
- simplify words and labels

## Phase 5: Trust and Visibility Layer

Goal: strengthen confidence between provider and farmer.

- shareable ledger summaries
- farmer-side visibility
- cleaner payment history
- settlement status clarity

## Phase 6: Operational Scale

Goal: move from one-provider proof to a repeatable product.

- multiple providers
- village and service segmentation
- better reporting
- local language expansion

## Phase 7: Platform Expansion

Goal: evolve from ledger tool to rural operations system.

- booking workflows
- reminders and alerts
- richer analytics
- broader farmer service ecosystem
